How to use project templates


To use a project template, you must create a project based on that template:

- Select *Projects* from the main navigation, and click ADD PROJECT.
You will now be shown a list of project templates that have been shared to your space. 

- Select the template you want to use.




- Give the project a name, and choose where the settings, non-sensitive variables, and template values will be stored.


- Click “Next” to select your versioning preferences for the template. 

Versioning preferences can changed later in project settings.
If the template has any pre-release versions, you will also be able to select between the following version types:
*Stable* -Intended for production use
*Pre-release* - These are intended for testing purposes only, and usually just by the creators of the templates. These are not recommended for production use.



- Select how to you'd like the template to update.
  
There are two options:
1. Accept minor changes 
   - This will automatically update the template whenever a patch or minor version is published.
   - Major versions will require a manual update.
2. Accept patches
   - This will only automatically update the template when a patch is published.
   - Minor or major versions will require a manual update.





Click *Create Project*, and you will be taken to the *Template values* page.

This is a list of parameters provided as part of the template, to allow you to customize the template to your needs. These are very similar to project variables - you can either provide a value directly, or use variable substitutions. They may have defaults provided, in which case they will be marked as optional.



Once you have provided these values, click *Create Release* to create a release of the project. This release will be based on the template, and is ready to deploy!
