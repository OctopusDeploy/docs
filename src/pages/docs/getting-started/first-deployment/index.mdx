Projects let you manage software applications and services, each with their own deployment process.

## Create deployment process

The next step is creating your deployment process. This is where you define the steps that Octopus uses to deploy your software. 

For this deployment, we will configure one step to print Hello World.

1. In the dialog, click **Thanks, got it**.

Octopus lands you in the process step template library.

2. In the **Featured** category, locate the Run a Script card and click **Add Step**.

:::figure
![Featured step templates](ADD IMAGE PATH)
:::

### Step Name
You can leave this as the default _Run a Script_.

### Script Source

You can source script files via three different methods:

* Inline script (default)
* Git repository
* Package

3. Select **Inline script** as your script source. 

:::figure
![Script source expander where users can select where to source scripts from](ADD IMAGE PATH)
:::

### Inline Source Code

4. Select an appropriate script language.
5. **Copy** the script below and **paste** it into the source code editor.

```
Write-Host "Hello, World!"
```

:::figure
![Inline source code expander where users type a script](ADD IMAGE PATH)
:::

## Release and deploy

### Create release

A release is a snapshot of the deployment process and the associated assets (packages, scripts, variables) as they exist when the release is created.

1. Click the **Create Release** button.
2. Click **Save**.

### Execute deployment

Deployments typically occur in a defined environment order (e.g. **Development âžœ Staging âžœ Production**), starting with the first one. Later, you can configure Lifecycles with complex promotion rules to accurately reflect how you want to release software.

2. Review the preview summary and when youâ€™re ready, click **Deploy**.
